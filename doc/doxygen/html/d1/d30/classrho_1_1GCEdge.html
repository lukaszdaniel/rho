Detailed Description

template<class T = RObject>
class rho::GCEdge< T >

Directed edge in the graph whose nodes are GCNode objects.

This class encapsulates a pointer from one GCNode to another, and carries out housekeeping required by the garbage collection scheme. The class name reflects the fact that these objects represent directed edges in the directed graph with the GCNode objects as its nodes.

Whenever an object of a type derived from GCNode needs to refer to another such object, it should do so by containing a GCEdge object, rather than by containing a pointer or reference directly.

Template Parameters
TGCNode or a type publicly derived from GCNode. This may be qualified by const, so for example a const String* may be encapsulated in a GCEdge using the type GCEdge<const String>.

Note
Why can't I specify the target in the constructor? Suppose that Foo, Bar and Baz are all classes derived from GCNode, and that a Foo object in effect 'contains' a Bar and a Baz. If it were possibly to initialize a GCEdge in its constructor, it would be tempting to implement the Foo constructor as follows: 
Foo()
     : m_edge1(new Bar), m_edge2(new Baz)
{}
But now consider what would happen if the call new Bar resulted in a garbage collection. Then the visitReferents() function of the object under construction may be called before the field m_edge2 has been initialized, i.e. when it still contains junk, and this will result in undefined behaviour, probably a program crash. This bug would remain latent until a garbage collection happened at precisely this point.
