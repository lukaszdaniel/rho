50 /* The Hershey vector fonts we support. Each character in a Hershey font
51  is an index into the glyph array in g_her_glyphs.cpp. Each
52  plHersheyFontInfoStruct includes these elements:
53 
54  (1) PS-style name for the font
55  (2) an alias for the font (for backward compatibility)
56  (3) Allen Hershey's original name for the font
57  (4) the characters in the font (an array of glyph indices, size 256)
58  (5) a typeface id (an index into the _ps_typeface_info[] array below)
59  (6) a font index (which font within the typeface this is)
60  (7) an `obliquing requested' flag (set if glyphs should be sheared)
61  (8) an `iso8859-1' flag
62  (9) a `visible' flag (false for the two Kana fonts,
63  which are only used internally) */

65 /* Each Hershey font below may contain up to 256 Hershey glyphs, each of
66  which is specified by a number that indexes into the array in
67  g_her_glyph.cpp. Only the ranges 0x20..0x7e and 0xa0..0xff are directly
68  accessible to the user.
69 
70  There are several sorts of hole in the Hershey fonts, i.e., entries in
71  the character arrays in these printable ranges which are not proper
72  glyphs. They are:
73 
74  0. undefined characters: thorn, eth, the guillemets, the mysterious
75  `currency' symbol, and the European 'euro' currency symbol. These are
76  written as UNDE, which is a glyph consisting of several horizontal
77  lines.
78 
79  1. the characters onesuperior, twosuperior, threesuperior, ordmasculine,
80  ordfeminine. These are written as 0, which means an empty glyph. That
81  is because they not implemented as conventional glyphs at all: the
82  corresponding entries in this table are never accessed. In the file
83  g_cntrlify.cpp these characters are mapped to sequences of control
84  sequences, which implement them as superscripts.
85 
86  2. the characters ae, AE, and germandbls (i.e. eszet), which (except in
87  Gothic-German) are also written as 0. That is because they are
88  `deligatured' in g_cntrlify.cpp, via a translation table in g_cntrlify.h.
89  Deligaturization maps them to the two-characters sequences "ae", "AE",
90  and "ss". The corresponding entries in this table are never accessed.
91 
92  3. the accented ISO-Latin-1 characters. These are written as ACC0,
93  ACC1, or ACC2, any of which signifies that they should be looked up in
94  the following table of accented characters, yielding an accent and a
95  character. They signify respectively that the accent should be
96  superimposed on the character, that it should be raised by 7 Hershey
97  units (for capitals), and that it should be raised by 7 units and also
98  displaced rightward by 2 units (for italic capitals).
99 
100  4. the small Katakana in the HersheyEUC font. These are written as KS
101  (i.e. Katakana small) + true glyph number. This means that they should
102  be isotropically compressed, with the baseline preserved.
103 
104 
105  In many Hershey fonts, the `nonprintable' range 0x80..0x9f is
106  nonprintable only in the sense that it is not directly accessible to the
107  user; it may contain ligatures, macros, and variant characters. In fact
108  the just-mentioned accents, used in the construction of accented
109  ISO-Latin-1 characters, are stored there. Our convention for the
110  0x80..0x9f range, for ISO-Latin-1 Hershey fonts, is:
111 
112  0200: ff
113  0201: fi
114  0202: fl
115  0203: ffi
116  0204: ffl
117  0205--0207: other ligatures (font-specific)
118  0210: acute accent
119  0211: grave accent
120  0212: dieresis
121  0213: circumflex accent
122  0214: circumflex accent for `i'
123  0215: tilde accent
124  0216: ring accent for `a'
125  0217: cedilla [NOT YET IMPLEMENTED]
126  0230: variant [final] form for `s'
127  0231: dotless i
128 
129  In the symbol fonts, we store miscellaneous symbols and astronomical
130  symbols in this range. In fact, in the symbol fonts the astronomical
131  symbols overflow to the other nonprintable range 0x00..0x1f
132  (the twelve zodiacal signs are stored there). */
