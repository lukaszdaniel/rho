31 /* This file contains many #defines and declarations of data structures.
32  More importantly, it contains declarations of all the Plotter methods.
33  They are declared differently, depending on whether we are compiling
34  libplot (signalled if NOT_LIBPLOTTER is #defined) or libplotter.
35 
36  In libplot, the plotter operations are implemented as global functions
37  that are members of the Plotter struct. They are set up differently for
38  the different types of Plotter; for example, the `openpl' slot in the
39  struct contains the method _g_openpl for generic [i.e. base] Plotters,
40  the method _m_openpl for MetaPlotters, etc. The files ?_defplot.c
41  contain the initializations that are used for the different types of
42  Plotter. In this file, if NOT_LIBPLOTTER is defined then each of these
43  many methods is declared as a global function.
44 
45  In libplotter, the different types of Plotter are implemented as
46  distinct classes, which are derived from the generic [i.e. base] Plotter
47  class. This file contains a great many #defines that are appropriate to
48  that situation. For example, _m_openpl is #defined to be
49  MetaPlotter::openpl if NOT_LIBPLOTTER is not defined. The MetaPlotter
50  class, like all other Plotter classes, is defined in plotter.h. */
