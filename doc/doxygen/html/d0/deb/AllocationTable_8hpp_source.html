43 /*
44  * Special-purpose hashtable implementation for tracking medium and large
45  * allocations.
46  *
47  * The table uses open addressing with quadratic probing to resolve
48  * collisions. This hashtable differs from dense_hash_table by
49  * allowing asymmetric insertion and searching - allocation ranges are
50  * inserted but the lookup function searches on a pointer so the search
51  * will look for a range covering the pointer.
52  *
53  * Another difference from most other types of generic maps is that this
54  * hashtable will map a single allocation to multiple hash buckets - the
55  * reason being that each hash bucket represents addresses that hash into
56  * that bucket, and all internal pointers of an allocation must be able
57  * to be found via the search() function.
58  *
59  * For hashing pointers, we shift away LOW_BITS bits from the pointer before
60  * hashing, and this limits the range of hash keys one allocation can hash
61  * to.
62  */
