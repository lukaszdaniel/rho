40 // The reference counting scheme uses an incremental stack scanning
41 // implementation for updating the reference counts of objects that have
42 // stack roots.
43 //
44 // The GC maintains a location on the stack known as the GC stack barrier, which
45 // has the property that all stack roots below the barrier have incremented
46 // their reference count, and none of the ones above the barrier have.
47 // At garbage collection time, only the locations on the stack between the
48 // barrier and the top of the stack need to be scanned, and the barrier can be
49 // advanced to reduce the amount of scanning that needs to occur in later
50 // collections.
51 //
52 // A GCStackFrameBoundary is a location on the stack where the barrier can be
53 // placed. To ensure correctness of the reference counting, it must be
54 // inserted only in locations where all the roots further down the stack are
55 // known to be immutable during its lifetime.
56 // The GCStackFrameBoundary's destructor automatically tests if the stack is
57 // being popped past the barrier, and if so, moves the barrier and updates the
58 // reference counts of the stack roots.
